# Contributing

Thank you for your interest in contributing to this repository.

## Getting Started

1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Install dependencies from `requirements.txt`.
4. Add tests for any new functionality.
5. Submit a pull request with a clear summary of changes.

## Code Style

- Keep changes focused and easy to review.
- Use descriptive commit messages.
- Do not include generated or temporary files in pull requests.

## Reporting Issues

If you find a bug or have a feature request, please open an issue using the templates provided in `.github/ISSUE_TEMPLATE`.
